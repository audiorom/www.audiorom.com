How to create a new message boards

[1] Create a new folder

Using Fetch create a new folder within:

/www.audiorom.com/talk

ie.

/www.audiorom.com/talk/new_board

and make the permission of this new folder

777 - read/right/execute for all (all selected)

using set permissions

[2] Copy files from example_board

Copy the contents of example_board into your new folder (don't edit the originals):

/www.audiorom.com/example_board

index.html
passwd.dat
seq.dat
arc (folder)
msg (folder)

[3] Edit index.html
Edit the following lines in your new copy of index.html:

(i) Replace title with the title of your new board 

<TITLE>AudioRom : Example</TITLE>

<TITLE>AudioRom : New Board</TITLE>
(ii) Replace headline with the headline of your new board 
<!-- name of board -->
<FONT FACE="VERDANA, ARIAL, HELVETICA" SIZE="4"><B>Example Board</B></FONT></FONT>

<FONT FACE="VERDANA, ARIAL, HELVETICA" SIZE="4"><B>New Board</B></FONT></FONT>
(ii) Replace the directory and name of your your new board

Edit the following line replacing the example folder::Board Name with that of your new board 

<!-- don't delete these two lines -->
<!-- they should be dir of the board::name of the board -->
<INPUT TYPE=HIDDEN NAME="bb_id" VALUE="example_board::Example Board">

<INPUT TYPE=HIDDEN NAME="bb_id" VALUE="new_board::New Board">

[4] Change permissions for the contents of your new folder

Change the pemission of your new folder new_board, and all it's contents, files and folders to

777 - read/right/execute for all (all selected)

using set permissions

[5] Edit Registry file, reg.dat

Open /www.audiorom.com/talk/reg.dat and add a line for your new board as follows: 

# Multi4M Registry
# Format: 'bb_dir::bb_name'
audio::Audio
beta::Beta
new_board::New Board

note: This line should be identical to the line you added to index.html in step 3

[5] Finally...

Add a link from the index page /www.audiorom.com/talk/index.html to your new board

Your new board should now been working, if it isn't check the permissions on all your new folders and files.

Finally be very, very carefully about modifying the new boards by hand once they are live
